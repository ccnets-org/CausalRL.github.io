    Release Notes


    v1.2.4

    • Released on Mar 3, 2024 view on GitHub

    • TD End Value Computation Enhanced

      Improved the calculation of the terminal value in Temporal Difference (TD) updates, particularly for trajectories that terminate early.

    • Adjustment in Learnable TD Weights

      Modified the weighting in the LearnableTD class, granting gamma parameters ten times the learning weight of lambda parameters. This adjustment prioritizes the optimization of discount factors, reflecting their critical role in the model's decision-making process and overall learning efficiency.

    • Refined Gradient Clipping Strategy

      Implemented a specialized gradient clipping approach for LearnableTD instances, applying clipping ranges exclusively to this class. This targeted strategy ensures optimal regularization of gradients, preserving model stability and facilitating nuanced control over the learning dynamics of critical parameters.

    • Model Saving Functionality Improved

      Resolved issues with the automatic best model saving mechanism during training. Models are now retained post-training completion, ensuring valuable progress is not lost and facilitating subsequent evaluation or continued training sessions.


    v1.2.3

    • Released on Mar 3, 2024 view on GitHub

    • Normalization and Weighting Adjustment

      Addressed an issue with the weighting of normalized sum rewards. Previously, the sequence weighting was divided by the TD_sequence_length(default 20), which exceeds the GPT_sequence_length(default 16), leading to diminished value loss or advantage and impeding learning motivation. This fix ensures rewards are appropriately weighted to motivate effective learning.

    • Learnable TD Class Update

      Adapted to use a lambda array aligned with the GPT_sequence_length (default 16) instead of the longer TD_sequence_length (20). This key change synchronizes lambda parameters with the sequence length in model predictions, enhancing Temporal Difference (TD) calculation accuracy by ensuring proper weighting of normalized sum reward sequences.

    • Sum Rewards Normalization Adjustments

      Refined the sum rewards normalization to concentrate on the GPT_sequence_length, moving away from the broader TD_sequence_length. This adjustment makes normalization directly relevant to the segments critical for TD updates, improving the consistency and relevance of normalized rewards.


    v1.2.2

    • Released on Mar 3, 2024 view on GitHub

    • Enhanced TD Return Calculations and Streamlined Computation

      Ensured dimensional consistency between expected_value (n+1) and sum_rewards (n), significantly improving the accuracy of subsequent operations. Furthermore, we've streamlined the computation process of expected values and sum of rewards. This includes refining the logic for trimming expected_value to precisely match the sequence length of sum_rewards and enhancing the clarity of our code through more concise commenting.

    • Flexible TD Sequence Lengths

      The introduction of td_seq_length enables precise calculations of TD returns, allowing for a more adaptable adjustment of sequence lengths for value estimation. This flexibility supports a dynamic range of temporal dependencies, optimizing the learning capabilities of the model.

    • Reverted Compensation Weights

      To address gradient deflation issues associated with padding, compensation weights in the masked_tensor_reduction function have been reverted.

    • Optimized Trajectory Adjustments

      By focusing on the differential segment (td_seq_length - gpt_seq_length), we've streamlined trajectory adjustments for TD calculations. This approach ensures accurate lambda returns calculation over the crucial end segment of the trajectory, enhancing both efficiency and precision.

    • Corrected Iteration Logic

      An essential correction has been made in the iteration logic for lambda returns calculation, ensuring it properly starts from the end of the sequence. This change addresses a crucial indexing mistake, enhancing the model's fidelity to TD principles.


    v1.2.1

    • Released on Mar 2, 2024 view on GitHub

    • Improved Backpropagation accuracy for learnable TD

      This enhancement allows for a more refined adjustment of these crucial parameters during learning.

    • Optimized Learnable TD Updates

      The learnable TD's update frequency has been modified to every 10 iterations. This optimization aims to enhance the efficiency of the training process without compromising the quality of learning.

    • Expanded Lambda Variable

      The lambda variable has been expanded to match a GPT sequence length, enabling individual optimization of each lambda value within the sequence. This allows for a more granular level of optimization, while gamma remains a singular value to maintain consistency across updates.

    • Flexible Learning Rate Adjustment

      Replaced the learning rate decaying system's min_lr with decay_rate_100k, offering a more intuitive and flexible approach to learning rate adjustment over training.

    • Gradient Consistency Improvements

      Removed compensation weights in masked_tensor_reduction function to address gradient deflation issues associated with padding parts. This change ensures the total tensor receives consistent gradients each step without artificially compensating for lessened weights of some states.


    v1.2.0

    • Released on Mar 1, 2024 view on GitHub

      • This release marks a significant milestone for the Causal-RL project, introducing the much-anticipated feature of auto-learning for the gamma and lambda parameters. With v1.2.0, we are proud to unveil our "One Click Causal RL" capability, streamlining the reinforcement learning process and enhancing adaptability across various environments.

    • Auto-Learning Gamma and Lambda

    v1.1.14

    • Released on Feb 29, 2024 view on GitHub

    • Improved Cooperative Error Calculations

      Adjustments have been made to halve the cooperative error calculations, addressing gradient scaling issues linked to double transitions minus one detached transition. This refinement ensures a more accurate balance between causal loss and value loss.

    • Refactored Value Computation

      The compute_values function has been divided into compute_expected_value and compute_advantage, clarifying the separation of logic and streamlining the calculation process.

    • Streamlined Advantage Normalization

      The previous advantage normalizer has been removed from the default implementation, simplifying the framework. Additionally, we've introduced Running_mean_sqrt as an optional advantage_normalizer to improve training stability.

    v1.1.11

    • Released on Feb 25, 2024 view on GitHub

    • Refined Reduction Strategy

      Refactored masked_tensor_reduction function to enhance flexibility and efficiency in handling different types of reductions: "batch", "seq", "cross", and "all". The function now accurately adjusts mean calculations based on the ratio of unmasked elements to total batch size for "batch" and "seq" reductions. For "cross" reduction, it ensures equal weighting of batch-wise and sequence-wise reductions, facilitating consistent learning signals across dimensions. The use of GradScaler has been optimized to adjust gradients appropriately, maintaining balanced contributions and equivalent learning impacts across reduction types.

    • Standardized Exploration Decay

      With the introduction of DECAY_UNIT_STEPS, we've standardized the exploration decay across different max_steps, ensuring uniform probability changes in padding mask sampling and maintaining proportional impact in varying training regimes.


    v1.1.10

    • Released on Feb 23, 2024 view on GitHub

    • Transition to "Cross" Reduction

      Transitioned the loss reduction strategy from "batch" to "cross" reduction for loss, optimizing rewards more effectively across batch and sequence dimensions.

    • GradScaler Integration

      To address potential gradient reduction with "cross" reduction, we've integrated GradScaler with a scale_factor of 1, preserving gradient magnitude and preventing loss during optimization.

    • Normalization Technique Adjustments

      Moving beyond the default running_absolute_mean, we're experimenting with running_mean_std for advantage normalization. We've also adjusted the update decay rate to 1e-3 for better balance and stability.


    v1.1.9

    • Released on Feb 22, 2024 view on GitHub

    • Refined Trajectory Sequences

      The TD computation in our GPT-based RL model has been fine-tuned for better accuracy and efficiency. Previously, trajectories were sampled at 1.5 times the GPT sequence length. In this update, we've adopted a more precise approach by adding an extra quarter to the original sequence length. This adjustment allows for a strategic division between causal training segments and TD computation, enhancing the model's capability to process extended sequences more effectively.

    • Enhancing Buffer System

      We have replaced the probability-based sampling method with a deterministic index selection method. This change is designed to address and mitigate issues related to non-repeatability that were observed with the previous sampling approach.

    • Enhanced Reproducibility and Consistency

      The shift to a deterministic method significantly improves the reproducibility of results across different training runs. This ensures that model behavior remains stable and predictable, which is crucial for research and development projects requiring consistent outcomes.

    v1.1.8

    • Released on Feb 21, 2024 view on GitHub

    • Enhanced TD Computation in GPT-based RL

      Introduced enhanced Temporal Difference (TD) computation utilizing extended trajectory sequences (gpt_td_seq_length). This update strategically divides trajectories sampled at 1.5 times the GPT sequence length into causal training and TD computation segments, overlapping to enhance accuracy through attention-based TD value computation.

      select_train_sequence and calculate_sequence_end_value methods added: Enable effective segmentation of extended trajectories for optimized learning. The front part of the trajectory is used for causal training, while the rear part, sharing some length with the front, is designated for precise TD computation. This shared segment approach significantly boosts accuracy and learning efficiency.

    v1.1.5

    • Released on Feb 18, 2024 view on GitHub

    • Lambda Adjustment Mechanism

      Transitioned the lambda adjustment model from an exponential to a linear model. This change ensures a more equitable distribution of weights across sequences, placing increased emphasis on recent sequences prior to backpropagation.

    • GPT Padding Length Strategy

      Enhanced the exploration strategy by adopting a linear increase in the sampling ratio for longer sequences. This adjustment works in tandem with the linear lambda model, facilitating a more strategic exploration of sequence lengths and improving learning efficiency from longer sequences.

    • Normalization Techniques

      Converted normalization techniques for state, reward, and sum_reward by implementing alpha values of 1e-5 for normalization of state, reward and sum_reward and 1e-4 for advantage normalization. The decay factor is set to 1 - alpha. This approach emphasizes recent experiences and reduces the impact of older data, enhancing the model's adaptability to new information.
