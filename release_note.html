    Release Notes


    v1.1.6

    • Released on Feb 19, 2024 view on GitHub

    • Unified Exploration Strategies

      Unified GPT padding and trajectory sampling strategies to ensure equal probabilities across sequences. This change promotes a more diverse exploration of sequences, allowing for fairer learning from a variety of experiences.

    • Learning Parameter Adjustments


    v1.1.5

    • Released on Feb 18, 2024 view on GitHub

    • Lambda Adjustment Mechanism

      Transitioned the lambda adjustment model from an exponential to a linear model. This change ensures a more equitable distribution of weights across sequences, placing increased emphasis on recent sequences prior to backpropagation.

    • GPT Padding Length Strategy

      Enhanced the exploration strategy by adopting a linear increase in the sampling ratio for longer sequences. This adjustment works in tandem with the linear lambda model, facilitating a more strategic exploration of sequence lengths and improving learning efficiency from longer sequences.

    • Normalization Techniques

      Converted normalization techniques for state, reward, and sum_reward by implementing alpha values of 1e-5 for normalization of state, reward and sum_reward and 1e-4 for advantage normalization. The decay factor is set to 1 - alpha. This approach emphasizes recent experiences and reduces the impact of older data, enhancing the model's adaptability to new information.

    • RL Configuration Adjustment


    v1.1.4

    • Released on Feb 16, 2024 view on GitHub

    • Dynamic Lambda and Sequence Sampling

      Introduced dynamic lambda to better balance immediate and future rewards based on exploration rate, adapting the learning strategy to the agent's current state.

      Modified the sample_padding_lengths function to improve sequence length sampling. Changed the adjusted_ratios calculation to be biased to the sampling towards longer sequences earlier in training, which is more beneficial for language model (LM) sequence learning. The update aims to avoid the uniform sampling of sequence lengths that previously hindered initial exploration effectiveness.

      advantage_lambda is now dynamically adjusted, starting from a base of 0.90. These changes refine the learning process, enhancing the model's ability to learn from longer sequences and adjust its strategy based on exploration progress.

    • RL Configuration Adjustment


    v1.1.3

    • Released on Feb 15, 2024 view on GitHub

    • Exploration Rate Decaying Strategy

      Updated the exploration decay mechanism for sequence sampling to achieve a controlled exploration rate, transitioning from a rapid to a more measured exponential decay.

    • Dynamic Sequence Weighting Before Backpropagation

      Implemented an exponential decay function for dynamic sequence weighting, replacing the previous linear combination method. This shift optimizes gradient calculations by balancing sequence weighting across the training process and exploration rate, rather than focusing solely on the latest state until the end of training.

    • Adjusted reverse_env Model Specification

      Adjustments made to minimize overfitting, thereby increasing the model's generalizability.

    • RL Configuration Adjustment


    v1.1.2

    • Released on Feb 14, 2024 view on GitHub

    • Conditional Parallel Processing

      A new mechanism within the rev-env network now allows for toggling between parallel and sequential processing of states. This capability is based on the current dropout configuration, resulting in a direct increase in computational speed.

    • Speed Improvement

      The implementation of parallel processing leads to a noticeable speed boost in state generation within the rev-env network. This enhancement streamlines both the training and evaluation phases, making the overall process more efficient.

    • Algorithm Logic Bug Fix

      A critical bug was corrected concerning the management of the estimated_value tensor. Previously, this tensor was not detached before being passed to self.revEnv for generating the recurred_state, leading to potential issues with inadvertent backpropagation. This fix ensures that the computational graph is properly managed, enhancing the algorithm's reliability and performance.

    • RL Configuration Adjustment


    v1.1.1

    • Released on Feb 13, 2024 view on GitHub

    • Parallel Processing Enhancements and Hyperparameter Tuning

      Parallel Processing with TorchScript: Integrated TorchScript's asynchronous execution capabilities for parallelizing the computation of reversed_state and recurred_state within the RevEnv module. Utilizes torch.jit.fork and torch.jit.wait for concurrent execution, drastically reducing processing time and boosting system efficiency. Ensures RevEnv module and its methods are TorchScript compatible to enable effective parallel processing.

    • Hyperparameter Adjustments


    v1.1.0

    • Released on Feb 13, 2024 view on GitHub

    • Optimized Gym Environment Setting

      Extensive benchmarks on four games(HalfCheetah, Hopper, Walker2D, Ant) with an optimized single setting.

    • Learning Rate Adjustment

      Adjustments to the default learning rate setting for optimal performance across benchmarks.

    • Environment-Specific Optimization

      Adjustments to gamma values in Ant and Hopper environments to optimize performance, demonstrating the model's adaptive capabilities.

    • Masked Exploration Improvement

      Logic improvements and a linear increase in padding length for exploration sampling.

    • MLAgent Environment Fix

      Fix for running issues in mlagent environments, optimizing control of exploration padding lengths and adjusting max_sample_weight for improved exploitation towards training end.



    v1.0.5

    • Released on Feb 2, 2024 view on GitHub

    • Refactor for Enhanced Loss Reduction and Boltzmann Exploration Adjustments

      Updated loss reduction method: To "cross" from "batch", combining batch and sequence dimension reductions each normalized by dividing by 2.

      Boltzmann Exploration parameters revised: Initial temperature (tau): Set to 1.0.

      Minimum temperature (min_temperature): Adjusted to 0.01.

      This update ensures robust initial exploration and balances learning, improving model adaptability and performance consistency across unseen environments.


    v1.0.4

    • Released on Feb 1, 2024 view on GitHub

      Adjusted RL and Boltzmann Exploration settings for enhanced compatibility with general gym environments.

      Updated discount_factor (gamma): From 0.993 to 0.99.

      Removed gradient clipping limit: Adjusted clip_grad_range from 1.0 to 2.0.

      Fine-tuned Boltzmann Exploration: Adjusted tau from 1.0 to 0.05 and reduced min_temperature from 0.1 to 0.005.


    v1.0.3

    • Released on Jan 31, 2024 view on GitHub

    • Update: Uniform Parameter Tuning Across All Gym Environments

      Advantage Normalization Adjusted: Transitioned from L1_norm to None for broader compatibility.

      Min Exploration Temperature Increased: From 0.01 to 0.1 to enhance exploration stability.


    v1.0.2

    • Released on Jan 31, 2024 view on GitHub

    • Exploration Decay Mechanism Enhancements

      Linear Scheduler for Exploration Decay: Transitioned to exclusively using a linear scheduler for exploration decay, streamlining the process for a more consistent and simplified exploration strategy.

      Removal of Exponential Decay: Previously integrated exponential decay within the BoltzmannExploration class has been removed due to observed training instability issues caused by rapid decay from the combined exploration rate and temperature exponential decay. Only linear exploration decay will be utilized to ensure stability and effectiveness in training.

      This update focuses on refining the exploration decay mechanisms to enhance the stability and performance of the Causal RL framework.


    v1.0.1

    • Released on Jan 31, 2024 view on GitHub

    • Minor Patch Update

      Adjustments to Library Settings: Fine-tuning for more effective reinforcement learning with GPT models.

      clip_grad_range: Adjusted from None to 1.0 to ensure a more stable gradient clipping mechanism.

      min_temperature: Increased from 0.01 to 0.02 for Boltzmann exploration, shifting slightly more towards exploitation in sequence length sampling for GPT inputs.

      Aimed at improving the stability and performance of causal-rl models across various environments.
